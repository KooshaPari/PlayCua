# Specifications

## Acceptance Criteria

- Add the sladge badge near the README title.
- Do not modify runtime code, workflows, or test harnesses.
- Preserve canonical checkout changes by working only in the isolated worktree.
- Record the work in session docs.

## Out of Scope

- Changing CUA execution behavior.
- Reconciling Cargo, spec, ADR, benchmark, fuzz, native test, or worklog work.
- Merging the prepared badge commit into the dirty canonical checkout.
