## 1. Executive Summary

PlayCua is a **Rust-based computer-use automation (CUA) framework** with a hexagonal architecture. It replaces the upstream `trycua/cua` VM layer with a native Rust binary communicating via stdio JSON-RPC 2.0. The project is **active** with significant recent churn (June 2026) and is currently undergoing a multi-phase merge from the deprecated `bare-cua` repository.

**Overall health:** Moderate — strong governance scaffolding, but gaps in test coverage, stale branch debt, and build timeout issues require attention before stabilization.

### 6.2 CI Observations

- **Duplicate CodeQL:** Both `codeql.yml` and `codeql-rust.yml` exist. They are nearly identical. Consolidate into one.
- **Phenotype-tooling dependency:** 3 workflows (`quality-gate.yml`, `fr-coverage.yml`, `doc-links.yml`) require a sibling `phenotype-tooling` repository that is not present in CI, so they `|| true` (no-op). This is intentional fallback behavior but means those checks are not actually running.
- **SHA pinning:** Governance workflow enforces full SHA pinning for third-party actions. Most workflows comply, but `secret-scan.yml` uses `gitleaks/gitleaks-action@v2` (tag, not SHA) and `pre-commit.yml` uses `actions/checkout@v4` (tag). These should be pinned.
- **Actions pinned:** `actions/checkout`, `dtolnay/rust-toolchain`, `Swatinem/rust-cache`, `EmbarkStudios/cargo-deny-action`, `rustsec/audit-check`, `ossf/scorecard-action`, `github/codeql-action`, `bnjbvr/cargo-machete`, `obi1kenobi/cargo-semver-checks-action`.
- **Concurrency:** All workflows use `concurrency` groups with `cancel-in-progress` for PRs and non-main branches. Good practice.
- **Timeouts:** Most workflows have `timeout-minutes: 30`; `codeql-rust.yml` has 360m.

### 6.3 Renovate & Dependabot

- **Renovate:** Enabled with `config:recommended`, schedules Monday 4am, groups by manager.
- **Dependabot:** Configured for cargo, npm, gomod, github-actions, docker with weekly cadence and grouped minor/patch updates.
- **Overlap:** Both Renovate and Dependabot are enabled. This may cause duplicate PRs. Recommend disabling one for GitHub Actions.
